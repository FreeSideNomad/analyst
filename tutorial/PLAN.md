# Tutorial plan

**Goal**: a comprehensive, self-serve tutorial published on GitHub Pages
that walks a new user through the product's happy-path acceptance
criteria end to end — every step either a UI action (described by its
visible labels) or a `make` target runnable from this folder.

## Principles

1. **Happy-path ACs are the script.** Each chapter names the features it
   exercises; each numbered step ends in the observable result the
   acceptance board asserts. If the tutorial can't be completed, a
   shipped AC is broken.
2. **Everything tutorial-related lives in `tutorial/`** — pages, the
   Makefile, the compose stack, generated data. The Pages workflow copies
   `tutorial/pages/` into the site at build time (single source of truth
   here; nothing duplicated under `docs/site`).
3. **UI first; `make` for the rest.** Anything a UI-only user cannot do
   (start the stack, seed Berka into Postgres, generate the cross-DB
   sample pair) is one make target, run from this folder.
4. **Real data only** — the same kits the acceptance boards use:
   `samples/` (curated CSVs/Excel incl. messy files and case-variant
   values), Pagila in the demo Postgres, the on-demand Ames gallery,
   the curated Berka bundle, and Berka seeded into a tutorial Postgres.

## Structure

```
tutorial/
  PLAN.md            this file
  Makefile           app / app-ml / dbs / berka-db / data / status / down / clean
  docker-compose.yml the ML-variant app + the Berka Postgres (one network)
  pages/             the published tutorial (copied into Pages at build)
    index.md         overview, prerequisites, quickstart
    01-ingest.md     files → profiles → meaning → links → normalization  (001,006,009,010,013)
    02-ask.md        Q&A, trust trail, charts, exports, dashboards, curation (003,014,015,016)
    03-databases.md  connect, in-place answers, cross-source, cross-DB, restart (005,007,008,017,011)
    04-models.md     guided single-table models on real Ames data (012)
    05-relational.md the ML variant: Berka bundle, three tiers, authoring on a connected DB (018,019)
  data/              generated by `make data` (gitignored)
```

## Chapter → AC map (happy paths)

| Chapter | Features | The AC moments the user re-enacts |
|---|---|---|
| 01 | 001/006/009/010/013 | messy CSV ingested + profiled; Excel ingested; catalog text; FK discovered & validated; workspace-aware re-catalog; case-variant rule proposed→approved→revoked |
| 02 | 003/014/016/015 | question→answer with trust trail; clarification; save as chart, re-run live; export CSV/Excel; curation form answered; correction; dashboard from a sentence; filter; cross-filter; edit by prompt; print preview |
| 03 | 005/007/008/017/011 | connect Pagila; in-place answer; file×DB join; two DBs joined locally (make data pair); restart → catalog + credentials survive |
| 04 | 012 | Ames arrives from the gallery; features proposed with reasons; curate; deterministic training; honest $ evaluation; predictions as a dataset |
| 05 | 018/019 | ML variant up; Berka bundle in one click; three-tier training reproduces published research; connect the seeded Berka Postgres (`make berka-db`); author a task from a question; adjust + confirm decisions; coin-flip canary; train; registry story names the source |

## Publishing

- `.github/workflows/pages.yml` gains: trigger on `tutorial/**`, and a
  pre-build step `cp -r tutorial/pages docs/site/tutorial` (workspace
  only — never committed into docs/site).
- The manual's index links to `/analyst/tutorial/`.

## Non-goals

- OAuth/workspaces (feature 004) needs external provider config — noted
  in the index as optional, not scripted.
- Screenshots: v1 is label-precise text; the manual carries imagery.
