# Feature 014 — Charts & data exports

> Promoted from roadmap item `charts-and-exports` (later/p3), deliberately
> sequenced BEFORE `exports-visualizations-dashboards` (later/p2): a
> dashboard widget IS a saved chart — this feature builds the unit the
> dashboard composes. Sequencing decision made under the in-session
> full-autonomy delegation (owner AFK), recorded here and in the handoff.

Roadmap note: "Save answers as charts (type inferred, overridable); export
result sets to CSV/Parquet/Excel."

Charter anchors: producing "exports, visualizations, and dashboards" is core
scope; all bulk computation local (exports are engine work, never model
work); trust trail travels with every visual.

Scope sketch (fixed at discover-acs):
- Chart-type override on any answer (stat/bar/line/table where sensible).
- Save an answer as a named chart: persists the question/plan + chart
  config per workspace; opening re-runs the query live; trust trail
  preserved; rename/delete.
- Exports: answer result → CSV/Excel; dataset → CSV/Parquet/Excel;
  generated by DuckDB locally.
