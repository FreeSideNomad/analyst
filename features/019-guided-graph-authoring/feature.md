# Feature 019 — Guided graph authoring

> PROMOTED 2026-07-19 (owner): full autonomy, same container-gated ATDD
> condition as 012/018. Owner decisions locked at promotion:
> 1. **Sources** — uploaded workspace tables AND connected relational
>    databases (the owner: "most likely use case for GNN"), handled
>    uniformly: both are DuckDB relations, so the engine gains ONE
>    workspace-sourced build mode covering both.
> 2. **Curated data, no hooks authoring** — the test corpus is curated
>    (already-decoded) Berka; the hooks.py-equivalent
>    inspection/transformation layer is explicitly out of scope. Data
>    arriving through analyst is already typed and normalized.
> 3. **Governance line unchanged** — the schema spec is generated by
>    committed code from the catalog (no LLM); the LLM authors ONLY
>    bounded declarative task decisions (entity, label SQL through
>    sql_guard, horizon, cutoffs, excluded outcomes), each a
>    plain-language confirmation in the UI; generated Python never exists.
> 4. **Charter disclosure** — training materializes a TRANSIENT LOCAL
>    build of connected-DB tables (federation's "nothing copied" is about
>    querying; the registry story and docs must say training copies
>    locally, still never leaving the box).

## Scope (the cut)

- **In:** workspace-sourced DatasetSpec generation (profiled types →
  spec types, validated 009 links → edges, date columns → time
  candidates, derived-table-free); engine build-from-relations mode
  (files + federated scanners); guided task authoring (agent-proposed,
  user-confirmed, cassette-replayed); structural honesty gates
  (auto-exclusion of label-query columns, shuffled-label canary,
  single-feature leak detector); Berka-in-Postgres seed script (demo
  compose DB); equivalence validation both arrival paths vs the 018
  curated reference; registry story naming the source and the local
  materialization; analyst:ml container e2e with the demo Postgres.
- **Out (later):** hooks-equivalent transformation authoring; temporal
  Home-Credit-style horizons beyond what task YAML already supports;
  Q&A-predictions integration; olist; GPU.

## Autonomy

`high` — full pipeline delegated in-session (2026-07-19); owner takes
over exploratory testing at the green container gate.
