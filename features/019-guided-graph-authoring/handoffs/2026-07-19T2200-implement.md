## Findings worth knowing

1. **The equivalence loop caught a real leakage-design gap**: 'payments'
   is post-outcome but unread by the label SQL — the mechanical hidden
   floor alone let the graph degrade to 0.63. Fix: the authoring turn
   carries the post-outcome judgment (outcome_columns) ∪ mechanical
   floor, users can only grow it. The recorded LIVE turn proposed the
   exact curated label SQL and (independently) 'status'.
2. **SQLite flattens dates to text** through the scanner — measured; the
   board's connected path uses a self-provisioned seeded Postgres so
   types stay faithful (also the owner's likeliest real case).
3. **Container defects found by the gate**: connect returns 201 (assert
   fixed); the ui Card component swallows aria-label (moved to inner
   node); the authored flow must build under the WRITABLE /data/ml-cache
   (the 018 ro cache mount pattern does not fit workspace builds).
4. Live model chose its own cutoffs (1997/1998) — sensible without
   seeing data; the adjust-before-confirm step (AC-3) is where curated
   parity is a user decision, and the board exercises exactly that flow.
