# Handoff — Feature 003: Natural-language Q&A over a dataset

## What shipped

The canned feature-002 Q&A stand-in is replaced by a real, confidence-gated
agentic planner in real mode, while fixtures mode keeps a deterministic
LLM-free path. **The wire contract (CONTRACT.md Q&A shapes) is unchanged** —
the frontend needed no contract-side changes.

Pipeline (real mode, default): repository records → sorted `QueryTable`
metadata (profiles + catalog descriptions, sorted capped samples) →
`QueryPlanner` through the existing `LLMGateway` (egress capped + logged;
only metadata crosses) → structured answer/clarify/abstain JSON → closed-world
SQL validation (`domain/query_validation.py`) → local DuckDB execution
(`engine/query.py`, capped result) → deterministically shaped `AnswerResult`
with the trust trail (assumptions, lineage, executed SQL).

Gating: clarify on ambiguity (AskQuestion, options round-trip via
`/api/query/{id}/respond` re-planning with the choice), abstain when
out-of-scope, abstain with disclosure when parse fails / confidence < 0.5 /
validation fails / execution errors. **Unvalidated SQL never executes.**
